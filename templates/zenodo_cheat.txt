  Notes:
  - "Central DB" reflects whatever repository the publication DB has on
    file — could be Zenodo, could be GitHub/OSF/etc. Recorded verbatim.
  - "Our Zenodo code" is what we track for the Zenodo deposit we
    created (or that we detected was already on file centrally). Use
    `oa action <pub_id> set_zenodo_code --code <id>` to override.
  - "Our Zenodo DOI" is the dataset PID we are collecting. It will almost
    never appear in the central DB. It is derived as 10.5281/zenodo.<code>
    once the code is set; until then it is blank — paste the minted DOI
    here while you work, then record the code to make it permanent.
